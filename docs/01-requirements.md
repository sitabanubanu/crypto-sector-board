# 01 - 产品需求

## 1. 用户画像
- 加密市场短线交易者
- 编程零基础，依赖 AI 协作开发
- 核心诉求：**快速判断资金在往哪个板块流动**，决定当日交易标的和方向

## 2. 核心使用场景
1. **开盘前扫一眼**：打开网页，3 秒内看清哪些板块强、哪些弱
2. **盘中异动感知**：板块突然加速涨/跌，系统自动标红提醒
3. **轮动判断**：对比近 3 天/7 天强弱变化，判断资金是否在板块间迁移
4. **持仓对照**：自己的仓位在 Treemap 上高亮，一眼看到自己站在哪里
5. **币种细节**：鼠标悬停看具体高低价、振幅、涨跌幅度

## 3. 功能列表

### 第一程 MVP（阶段 0-6，本期交付）
- [ ] 自定义板块清单（手工维护 `data/sectors.json`）
- [ ] 每小时自动抓取行情（GitHub Actions cron，不是每天一次）
- [ ] 计算单币指标：最高、最低、振幅、收益率、波动率
- [ ] 计算板块指标：按市值加权平均
- [ ] Treemap 嵌套渲染（板块外层 + 单币内层）
- [ ] 颜色编码涨跌（红涨绿跌、深浅表幅度）
- [ ] 鼠标悬停 Tooltip 显示单币详情
- [ ] 顶部 Header 显示数据抓取时间 + 数据新鲜度（"X 分钟前"）
- [ ] Vercel 公网部署，手机也能打开

### 第二程 交易增强（阶段 7-10）
- [ ] **多时间维度对比**：3 天 / 7 天 / 30 天 板块强弱热力图
- [ ] **板块轮动检测**：自动标注"资金正从 X 板块流向 Y 板块"的信号
- [ ] **异动报警**：单币或单板块振幅/涨跌超过历史均值 2 倍时自动高亮
- [ ] **BTC 主导率指示器**：页面角落显示 BTC 市值占比变化趋势
- [ ] **成交量加权**：除市值外，把 24h 成交量也纳入板块热度计算

### 第三程 个人化（阶段 11-13）
- [ ] **持仓标记**：在 `data/sectors.json` 里标注自己持有的币，Treemap 上加星标边框
- [ ] **板块相关性矩阵**：14 个板块之间的涨跌相关性热力图，辅助分散化
- [ ] **Telegram 推送**：每小时抓完后自动发一条摘要到你的 Telegram
- [ ] **移动端专用视图**：手机打开自动切换为列表模式（非 Treemap）

### 长期探索（阶段 14+）
- [ ] 社媒情绪聚合（Twitter/Reddit 上各板块讨论热度）
- [ ] K 线点击下钻
- [ ] 网页内编辑板块清单的 UI
- [ ] 模拟回测：如果上周按轮动信号调仓，收益率会是多少

## 4. 非功能需求
| 维度 | 目标 |
|---|---|
| 数据新鲜度 | 每小时更新一次（免费层极限） |
| 性能 | 首屏 < 2s，Treemap 交互流畅 |
| 可用性 | Vercel 99% 可用即可（个人工具） |
| 可维护性 | 板块清单纯数据维护，不改代码 |
| 可观测性 | GitHub Actions 失败 → Telegram 通知 |
| 兼容性 | 桌面 Chrome/Edge/Safari + 手机浏览器 |

## 5. MVP 范围（本期交付）
- ✅ Treemap 主屏（昨日完整快照 + 实时性标记）
- ✅ 板块清单自定义
- ✅ 每小时自动抓取（GitHub Actions）
- ✅ Vercel 公网网址
- ✅ 数据新鲜度显示
- ❌ 多时间维度对比（第二程）
- ❌ 轮动信号 / 异动报警（第二程）
- ❌ 持仓标记 / Telegram 推送（第三程）

## 6. 与原版的区别
原 MVP 只做"每天抓一次 + 一张静态 Treemap"。新版 MVP 在此基础上增加了：
- **抓取频率**：从每天 → 每小时（交易时段信息密度大幅提升）
- **数据新鲜度**：页面上显示"X 分钟前更新"
- **后续路线清晰**：三条程线各有明确的交易价值，不再只是"好看"
